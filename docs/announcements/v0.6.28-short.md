# Vault-for-LLM v0.6.28

This release makes Supabase setup easier for humans and agents.

Highlights:

- `vault setup-agent` now supports setup language output with `--language en|zh-Hant|zh-CN`.
- Manual interactive setup asks for the setup language; non-interactive agent installs can pass `--language` without adding another prompt.
- Supabase setup now has `--supabase-setup none|simple|advanced`.
- Simple setup is the recommended default: use Supabase only for cross-host, team, Coze, n8n, or hosted-agent memory sharing.
- Advanced setup is reserved for RLS, multi-agent allow-lists, sensitivity tiers, and read-only hosted access.
- Added `docs/supabase_setup.md` with minimal schema, first-sync steps, and RLS guidance.

Vault remains local-first. Supabase is still optional, and local `vault.db` stays the source of truth.

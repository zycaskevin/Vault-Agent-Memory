# Vault-for-LLM v0.6.105

This release makes the multi-Agent automation handoff easier to start from.

`vault automation handoff` now surfaces the latest fleet-health panel before the
individual cycle or inbox handoff when `reports/automation/fleet-health-latest`
exists.

```bash
vault automation fleet-health --write-health
vault automation handoff
```

The CLI prints fleet health first so the next agent sees shared automation
health before task-specific instructions.

For MCP and JSON consumers, the selected handoff remains in `content`. The
shared panel is exposed separately as `fleet_health_content`, keeping existing
agent integrations stable.

Safety boundary: this is read-only. It only reads existing
`reports/automation` artifacts and does not read private memory, transcript
contents, raw candidates, or raw feedback reasons.

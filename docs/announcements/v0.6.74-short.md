# Vault-for-LLM v0.6.74

This release turns the automation cycle workspace into a more useful daily
handoff for the next agent.

## What changed

- `cycle-latest.json` now includes:
  - `priority_brief`
  - `suggested_next_tasks`
  - `agent_start_prompt`
- `cycle-latest.md` now has dedicated sections for:
  - Priority Brief
  - Suggested Next Tasks
  - Agent Start Prompt

## Why it matters

An agent should not need to read every automation report before doing useful
work. The handoff now starts with what matters most, what to do next, and the
safety rules for taking over the memory cycle.

The safety boundary is unchanged: the workspace still hides raw candidate
content, lists transcript exports as metadata-only paths, and reminds agents not
to auto-promote candidates or hard-delete memory.

## Try it

```bash
pip install "vault-for-llm[mcp]==0.6.74"

vault automation cycle \
  --project-dir /path/to/project \
  --write-workspace \
  --include-transcripts
```

Then start from:

```text
reports/automation/cycle-latest.md
```

# Vault Agent Memory v0.7.31

v0.7.31 ships the Central Memory Station foundation for multi-agent,
multi-host, and multi-device memory.

## What changed

- Added the daily Central Memory Station CLI surface:
  `vault start`, `vault memory-sync`, `vault memory-review`,
  `vault memory-lifecycle`, and `vault ops`.
- Added a trusted-host sync worker with generated cron, LaunchAgent, and n8n
  templates for 30-60 minute synchronization.
- Added Supabase Central Memory Station tables and a self-hosted central
  candidate inbox.
- Added Remote Server and Gateway endpoints for central candidate status,
  submit, and pull flows.
- Kept remote writes candidate-first. Active memory is still reviewed and local
  SQLite remains the local source of truth.

## Install

```bash
pip install --upgrade "vault-for-llm[mcp,supabase]==0.7.31"
```

## First check

```bash
vault start
vault memory-sync status --json
vault memory-sync run-once --dry-run --json
```

# Vault Agent Memory v0.8.0

v0.8.0 promotes the Central Memory Station foundation from release gate to
daily-use runtime for agent memory.

## What changed

- Added the daily-loop runtime status surface for scheduled automation,
  candidate queue health, human-review cards, and remote sync health.
- Added a pure report refresh path so operators can rebuild the latest
  daily-loop report without creating new candidates.
- Hardened Supabase Document Map sync by translating local SQLite knowledge IDs
  to remote Supabase UUIDs before node and claim upserts.
- Treated dry-run sync reports as previews rather than completed remote syncs.
- Added trusted-host service-role sync support for controlled local operators.

## Install

```bash
pip install --upgrade "vault-for-llm[mcp,supabase]==0.8.0"
```

## First check

```bash
vault daily-loop status --json
vault remote status --json
vault daily-loop report --refresh --write-report
```

# Vault Agent Memory v0.9.0

v0.9.0 is the Public Beta / Developer Preview release line for governed
multi-agent memory.

The core model stays conservative: local SQLite/Markdown remains the source of
truth, same-machine agents can share one trusted Vault, remote agents can read
approved memory and submit candidates, and only a trusted sync host should hold
service-role credentials or promote official memory.

## What changed

- Added the policy-aware remote semantic read chain for reviewed safe-summary
  embeddings, safe preview results, and bounded snapshot reads by `read_handle`.
- Added Gateway / Remote Server HTTP endpoints for the remote semantic read
  chain when explicitly enabled.
- Hardened governed-auto Dream queue convergence so low-risk metadata/dedup
  queue noise can close without hiding freshness, convergence, consolidation,
  sensitive, or strategic review work.
- Updated the public README, install prompts, landing snippets, installer
  defaults, integration docs, and release notes to frame the release as Public
  Beta / Developer Preview.

## Install

```bash
pip install --upgrade "vault-for-llm[mcp,supabase]==0.9.0"
```

## First check

```bash
vault --version
vault daily-loop status --json
vault remote status --json
vault vector-index central-status --json
vault daily-loop report --refresh --write-report
```

## Known limitations

- Remote or hosted agents should use anon/scoped credentials for approved reads
  and candidate submission only; service-role keys belong on the trusted sync
  host.
- Central vector search indexes reviewed safe summaries/previews as a derived
  read layer. It does not make Supabase the active-memory source of truth.
- Semantic query text may be sent to the configured embedding provider when
  remote semantic search is enabled.
- Automation remains candidate-first and report-first. Governed-auto can close
  routine low-risk metadata/dedup review noise, but freshness, convergence,
  consolidation, sensitive, and strategic work remains visible for review.
- LoCoMo / LongMemEval claims are limited to retrieval-only source-hit probes
  unless a comparable final-QA answerer and judge run is published.

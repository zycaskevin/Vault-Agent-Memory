# Vault-for-LLM v0.6.73

This release makes the scheduled memory cycle easier for both humans and agents
to pick up.

## What changed

- `vault automation cycle --write-workspace` now writes two files:
  - `reports/automation/cycle-latest.json` for tools and automation.
  - `reports/automation/cycle-latest.md` for humans and the next agent.
- The Markdown handoff summarizes:
  - candidate review queue
  - metadata-only transcript paths
  - curation policy hints
  - safety flags
  - next action
- Raw candidate content and transcript contents stay hidden.

## Why it matters

The automation loop can now leave a small daily handoff without forcing the
next agent to read full reports first. This keeps the memory system moving
toward progressive disclosure: summary first, bounded evidence next, raw source
only when needed.

## Try it

```bash
pip install "vault-for-llm[mcp]==0.6.73"

vault automation cycle \
  --project-dir /path/to/project \
  --write-workspace \
  --include-transcripts
```

Then open:

```text
reports/automation/cycle-latest.md
```

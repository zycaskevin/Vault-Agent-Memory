# Vault Agent Memory v0.10.0

v0.10.0 is the memory-foundation release for Vault Agent Memory.

The product boundary is now clearer: Vault is a local-first, backend-agnostic
memory governance layer for AI agents. It still works standalone through CLI,
MCP, Gateway, and local SQLite. It can also act as the governed memory
foundation underneath other agents and memory frameworks.

## What changed

- Added backend-agnostic positioning around the Vault Governance Contract and
  Memory Provider Interface.
- Added deployment-mode guidance for Local Vault, Trusted Local Central Memory
  Host, Supabase Adapter, and future Vault Cloud.
- Added the Self-host Central Memory Host specification and operator runbook.
- Added the draft Vault Memory API specification for create, search, read,
  update, delete, audit, timeline, and future sync/link/promote surfaces.
- Added the first runtime Memory Provider Interface and default SQLite provider
  facade, surfaced through Gateway health and OpenAPI.
- Added provider-read adoption metadata for `/memory/search` and `/memory/{id}`
  while keeping the legacy Gateway policy gate authoritative by default.
- Added opt-in provider-backed `/memory/search` and `/memory/{id}` preview
  result adapters through `result_adapter=provider`.
- Added metadata-only provider adapter parity reporting and
  `vault memory-api parity-report`.
- Added provider adapter default-promotion criteria before any future default
  Gateway authority switch.
- Updated English, Traditional Chinese, and Simplified Chinese README language
  so all three explain the same standalone/foundation and provider-preview
  boundaries.

## Install

```bash
pip install --upgrade "vault-for-llm[mcp,supabase]==0.10.0"
```

## First check

```bash
vault --version
vault daily-loop status --json
vault remote status --json
vault vector-index central-status --json
vault daily-loop report --refresh --write-report
```

## Safety boundary

- Remote or untrusted writes remain candidate-first.
- Provider-backed Memory API adapters remain opt-in preview paths.
- Default `/memory/search` and `/memory/{id}` authority remains the legacy
  Gateway policy-filtered path until provider parity is proven.
- Provider-backed search/read adapters do not return provider raw rows or raw
  memory content.
- The parity report does not return raw memory content or raw query text.
- Supabase remains an optional cloud adapter and reviewed read copy plus
  candidate inbox, not an active multi-master memory database.

## Known limitations

- Remote Semantic Search is disabled by default. If enabled without a local or
  otherwise trusted provider override, query text is sent to OpenAI by default.
- Vault Cloud is a future managed backend for the same governance contract, not
  a replacement for local, self-hosted, or Supabase deployment modes.
- Benchmark language remains retrieval-only source-hit evidence unless
  comparable final-QA answerer and judge runs are published.

# Vault-for-LLM v0.6.51

This release makes memory automation easier to install and schedule.

## Highlights

- `vault setup-agent` can now generate report-first memory automation templates:
  cron, macOS LaunchAgent, and n8n.
- Scheduled automation stays safe by default. It runs reports unless the user
  explicitly adds `--automation-apply`.
- Explicit `--project-dir` handling now respects empty target directories
  instead of climbing to a parent vault.
- The docs now include a practical MCP tool reference for profile selection,
  per-tool examples, and common agent mistakes.

## Example

```bash
vault setup-agent \
  --non-interactive \
  --agent automation-agent \
  --scope shared \
  --agent-project-dir ~/Vaults/project-memory \
  --features core,mcp,memory_agents \
  --automation-schedule all \
  --automation-mode balanced \
  --json
```

Review the generated `agent-install/README-memory-automation.md` and
`automation_policy.yaml` before enabling scheduled jobs.

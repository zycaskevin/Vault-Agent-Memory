# Vault-for-LLM v0.6.103

This release makes scheduled memory automation easier to trust at a glance.

When `setup-agent` generates cron, LaunchAgent, or n8n memory automation
templates, the scheduled job now writes the learning-health dashboard after the
automation pass and inbox handoff:

```bash
vault automation learning-health --write-health
```

That creates:

- `reports/automation/learning-health-latest.json`
- `reports/automation/learning-health-latest.md`

The report is intentionally short and safe: it shows whether the learning loop
is in cold start, healthy, needs watching, or needs review. It does not expose
raw feedback reasons, raw candidate content, or any new promotion authority.

# Vault-for-LLM v0.6.108

This release hardens the multi-Agent write boundary.

## Highlights

- MCP now rate-limits tool calls in process.
- Direct MCP writes into shared/public, private, high-sensitivity, or restricted memory require an explicit `agent_id` and matching `allow_*` flag.
- The privacy gate now warns on prompt-injection-like memory content, Taiwan phone/ID patterns, high-entropy tokens, and Base64-encoded sensitive content.

## Why It Matters

Vault can be shared by many agents and runtimes, but shared memory should not become a free-for-all. This release keeps the local-first path compatible while making broader writes deliberate.

## Upgrade

```bash
pip install -U "vault-for-llm[mcp]==0.6.108"
```

For shared MCP deployments, review any direct `vault_add`, `vault_memory_propose`, or `vault_memory_promote` calls that write outside low-sensitivity project memory.

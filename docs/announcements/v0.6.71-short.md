# Vault-for-LLM v0.6.71

This release adds a compact cycle workspace for the memory automation loop.

## What Changed

- Added `vault automation cycle --write-workspace`.
- The cycle can now write `reports/automation/cycle-latest.json`.
- Added cycle workspace options:
  - `--workspace-path`
  - `--inbox-limit`
  - `--include-transcripts`
  - `--transcript-limit`

## Why It Matters

The automation loop now has a clearer next-agent handoff:

```text
automation cycle -> cycle-latest.json
  -> candidate_review
  -> transcripts_to_capture
  -> curation_policy
```

The workspace is compact and conservative. It hides raw candidate content by
default, transcript discovery remains metadata-only, and the cycle still does
not promote candidates, hard-delete memory, or bypass privacy/access policy.

## Try It

```bash
pip install "vault-for-llm[mcp]==0.6.71"

vault automation cycle \
  --project-dir ~/Vaults/my-project \
  --write-workspace \
  --include-transcripts \
  --pretty
```

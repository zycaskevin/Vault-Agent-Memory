# Vault Agent Memory v0.10.1

v0.10.1 is a focused safety patch for the Vault Agent Memory Public Beta.

## What changed

- Private-memory read policy now activates when `include_private=True`, even
  if the caller omitted `agent_id`. Without a verified identity, private
  memory remains unreadable.
- Gateway token comparisons now use `hmac.compare_digest`.
- Provider-backed trusted local updates reject protected identity fields and
  invalid lifecycle status transitions.
- Read-policy callers can opt into an explicit lifecycle status allow-list.

## What did not change

- SQLite and Markdown remain the local source of truth.
- Remote or untrusted writes remain candidate-first.
- Provider-backed result adapters remain opt-in preview paths.
- Remote Semantic Search remains disabled by default.
- Central vectors remain a derived read layer, not active-memory authority.

When Remote Semantic Search is deliberately enabled, Gateway startup already
prints the active query embedding provider and model. It also prints a privacy
warning when query text will be sent to OpenAI or another external provider.

## Install

```bash
pip install --upgrade "vault-for-llm[mcp,supabase]==0.10.1"
```

After installation:

```bash
vault --version
vault quickstart
vault security doctor
```

Expected version: `vault-for-llm 0.10.1`.

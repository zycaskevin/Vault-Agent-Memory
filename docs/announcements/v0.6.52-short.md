# Vault-for-LLM v0.6.52

This release makes policy-based memory automation more reviewable and safer for
multi-agent vaults.

Highlights:

- `vault automation run` reports now include an `action_ledger` with per-memory
  before/after status, operation, risk, and reason.
- Automation reports also include a `dry_run_diff`, so agents can summarize what
  would change, what was applied, and what was skipped.
- Default automation policy now protects `scope: private` and
  `sensitivity: high|restricted` memories from routine TTL archival. These rows
  remain active and appear as `skipped_policy`, even with `--apply`.
- Human-readable CLI output now shows skipped-policy counts, diff summaries, and
  ledger entries.

Smoke test:

```bash
pip install "vault-for-llm[mcp]==0.6.52"
vault automation run --project-dir /path/to/project --apply --json
```

Expected behavior: low-sensitivity expired project memory may be archived when
policy allows it; private or high-sensitivity expired memory is reported for
review and left untouched.

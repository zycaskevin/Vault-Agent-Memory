### 3. Python version boundary

The same clean install failed when the virtual environment was created with the
macOS system Python 3.9.6:

```text
ERROR: Package 'vault-for-llm' requires a different Python: 3.9.6 not in '>=3.10'
```

This is correct package behavior, not a Vault runtime bug. For external users,
the install guide should keep making the Python 3.10+ requirement visible and
Agents should check Python version before installing.

## Remaining Polish Before A Broader Public Push

### P1: Keep Python version guidance very visible

Many ordinary macOS users still have an old system Python. The Agent install
path should explicitly check Python and say:

> Vault needs Python 3.10+. I will install or use a supported Python runtime
> before installing Vault.

### P2: Prefer rendered demo pages over raw Markdown links

Raw Markdown is fine inside GitHub or docs sites, but local browser navigation
can be blocked or shown as plain text depending on environment.

Follow-up status: addressed by adding `docs/landing/demo.html` and making the
landing page's primary demo links point to that rendered HTML page. The Markdown
runbook remains available as a secondary technical reference.

Completed by:

- adding a small rendered demo HTML page, and
- making the landing page's primary demo link stay inside `docs/landing/` and
  show the three-agent flow visually, with the Markdown runbook as a secondary
  technical link.

### P2: Keep first-run language and consumer mode coupled

The public path is now understandable because it says:

- choose language,
- choose independent/shared memory,
- optionally connect Obsidian or Supabase,
- choose daily report time.

Do not expose the full CLI command surface to ordinary users unless they ask.

## Release Gate Decision

**Pass for Agent-assisted public release.**

The current package can be released as long as the public announcement frames it
as:

- Agent Memory Governance, not another RAG database;
- install through an Agent or guided setup;
- daily report as the human review surface;
- low-risk automatic memory with reviewable governance boundaries.

Not yet recommended as a "zero-context, no-Agent, no-CLI" consumer app. That
requires a more complete hosted or desktop onboarding surface.

# v0.7.28 Onboarding, Safety, and P3 Closeout

Date: 2026-07-03

This closeout records the post-v0.7.28 public-preview hardening pass across
first-run onboarding, safety polish, remote candidate integrity, and demo depth.
It is not a new release note by itself; it is the evidence trail for deciding
whether the next patch release is ready.

## Current Product State

The current public entry is stronger than v0.7.28's initial publication:

- A new user can start with `vault quickstart` rather than reading every
  `setup-agent` flag.
- Gateway startup and HTTP responses have clearer safety posture.
- Dry-run expectations are tighter for Obsidian import.
- Export now has a symmetric path for getting memory out of Vault.
- Remote candidate integrity can rotate HMAC keys without exposing secrets.
- The public demo now has follow-up stories for first-run, automation, and
  remote candidate sync.

This still should not be marketed as a zero-learning consumer app. The strongest
position remains agent-assisted onboarding for builders and operators who want a
governed memory layer across agent runtimes.

## Remaining Risks

- No PyPI package was published from this closeout. The clean install used the
  local repository through pip, not the public index.
- The demo scenario guides are generated artifacts and runbooks; they are not
  full live end-to-end remote infrastructure demos.
- Remote candidate HMAC rotation is implemented at the application/template
  level, but operators still need to deploy the updated Supabase SQL before
  relying on `hmac_key_id` in existing projects.
- The system Python on macOS can still be too old; release docs should keep
  recommending an explicit Python 3.10+ environment.

## Recommendation

This pass is ready for the next patch-release decision if the release owner
wants to publish these changes. Before publishing, run the normal release
workflow:

1. bump version and release notes,
2. build wheel/sdist,
3. install the built wheel in a clean Python 3.10+ environment,
4. run `vault quickstart --non-interactive --json`,
5. run `vault demo agent-governance --json`,
6. confirm `vault remote hmac-keys --json` does not print secrets.

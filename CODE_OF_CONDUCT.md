# Code of Conduct

Vault-for-LLM is a project for Agent-assisted builders who care about memory
governance, privacy, and reliable automation. The community should be practical,
welcoming, and safe for people with different levels of Agent experience.

## Expected Behavior

- Be respectful and assume good faith.
- Keep feedback specific, actionable, and tied to the code, docs, or user path.
- Help new contributors find a smaller first step when a change is too broad.
- Protect privacy. Do not ask people to share real secrets, private chats,
  customer records, medical data, or production vault exports.
- Be clear when a suggestion is experimental, security-sensitive, or likely to
  change a public contract.

## Unacceptable Behavior

- Harassment, insults, threats, or sustained personal attacks.
- Publishing private information without explicit permission.
- Posting secrets, tokens, private logs, or sensitive personal data.
- Dismissing reports because a contributor is new to Agent tooling.
- Repeatedly pushing a major architecture change after maintainers ask for an
  issue or design discussion first.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues; reject PRs; or
block participation when behavior makes the project unsafe or unproductive.

If you need to report a conduct concern, open a GitHub issue with only the
minimum non-sensitive context needed, or contact the maintainer through the
public repository profile.

## Notes

Security reports and private-data accidents should focus on safe reproduction
and redaction first. Do not paste raw secrets or private vault contents into a
public thread.

Four steps

From application to public record.

Register a track

Open the GitHub form, choose a track and provider, describe the environment, and disclose affiliations.

Confirm the 14-day window

A maintainer confirms the seat and start date in the issue. Registration alone does not make an operator part of the cohort.

Run and retain raw evidence

Run preflight first. Preserve machine-readable artifacts, versions, commit SHA, deviations, and redacted logs.

Submit any honest outcome

Validated, Rejected, Environment blocked, or Protocol issue found can all close the challenge. Missing data never becomes zero.
